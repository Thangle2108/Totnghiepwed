# Contributing to Totnghiepwed

Thank you for taking the time to contribute! This project aims to be a professional Laravel-based web application.

## Coding Standards

- Follow **PSR-12** coding guidelines for PHP.
- Use 4 spaces for indentation, no tabs.
- Keep functions and classes well documented with PHPDoc.
- Write clear commit messages in the present tense.

## Pull Request Workflow

1. Fork the repository and create a feature branch.
2. Ensure your branch is up to date with `main`.
3. Run all tests and linting tools locally.
4. Open a Pull Request describing your changes.
5. Participate in code review and address feedback.
6. Once approved, your PR will be merged.

## Reporting Issues

- Use the GitHub Issues tracker.
- Provide steps to reproduce, expected results, and actual behavior.
- Include environment details such as PHP and Laravel versions.

# Đóng góp vào Totnghiepwed

Cảm ơn bạn đã quan tâm đóng góp cho dự án! Đây là ứng dụng Laravel chuyên nghiệp phục vụ bán hàng quần áo và hình ảnh.

## Quy chuẩn mã nguồn

- Tuân thủ tiêu chuẩn **PSR-12** cho mã PHP.
- Thụt lề 4 dấu cách, không dùng tab.
- Viết chú thích hàm và lớp bằng PHPDoc.
- Tin nhắn commit rõ ràng ở thì hiện tại.

## Quy trình Pull Request

1. Fork kho mã và tạo nhánh tính năng.
2. Đồng bộ nhánh của bạn với `main`.
3. Chạy toàn bộ kiểm thử và công cụ lint.
4. Mở Pull Request mô tả thay đổi của bạn.
5. Tham gia phản hồi mã và sửa theo góp ý.
6. Khi được chấp thuận, PR sẽ được gộp.

## Báo cáo lỗi

- Sử dụng trình theo dõi Issue trên GitHub.
- Cung cấp bước tái hiện, kết quả mong muốn và thực tế.
- Thêm thông tin môi trường như phiên bản PHP và Laravel.
